Installation of mxlib.

Introduction

These are instructions for installing the "mxlib" c/c++ library. This is my personal library which I have built up over many years. Along the way I have tried to keep it organized and documented. I have installed it on several different linux systems.

2017-07: I'm engaged in a fairly major reorganization and refactor. So right now not everything will work out of the box.

Basic Dependencies

  1. You need a c++11 compatible compiler (or c++14, or –std=c++0x in g++ works)
  2. You need the GNU Scientific Library (GSL) (this is probably obtainable from the linux distro package manager, but I recommend building yourself)
  3. You need the boost c++ library (this is probably obtainable from the linux distro package manager)
  4. You need cfitsio (http://heasarc.gsfc.nasa.gov/fitsio/)
  5. You need a BLAS and LAPACK. An excellent option is ATLAS, see my notes on installing ATLAS.
  6. You will need the levmar least squares library to use the data fitting tools.
  7. You need the FFTW library, with at least version 3.3.5.
  8. You need the IAU sofa libary.
  9. You need the Eigen matrix library (http://eigen.tuxfamily.org/index.php). I am currently on 3.3.4.

The relevant header files for each library need to be accessible to the compiler, either in a standard location or in one you specify when compiling your programs.

Build and Install

Follow these steps:

  1. Checkout mxlib from https://github.com/jaredmales/mxlib
  2. In the mxlib folder, possibly edit the Makefile. It probably isn't necessary. If it is, it is probably only necessary to edit INSTALL_PATH. But you can edit all of the following if needed:
    INSTALL_PATH = $(HOME)
    INCLUDE_PATH = $(INSTALL_PATH)/include/mx
    LIB_PATH = $(INSTALL_PATH)/lib
    BIN_PATH = $(HOME)/bin
  3. Once that's done, type make, then make install
